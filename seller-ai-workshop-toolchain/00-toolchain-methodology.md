# Методология инструментального воркшопа для селлеров

**Навигация:** [← README](../README.md) | [← Первый урок](../ai-agents-seller-workshop-lesson-01.md) | [Далее: воркшоп 1](workshop-01-data-api-storage.md)

## Главный разворот

Курс должен быть не про то, что преподаватель берет отчет участника и анализирует его руками вместе с AI.

Правильная суть:

```text
Мы учим селлера строить собственную AI-инфраструктуру:
получать данные -> сохранять -> приводить к понятному виду -> давать агенту контекст -> запускать регулярные проверки -> получать выводы и действия.
```

Ценность не в одном разборе отчета, а в том, что участник понимает, как самому настроить систему под свой бизнес.

## Что должны научиться делать участники

- Подключать источники данных: API маркетплейса, CSV, Excel, Google Sheets, ручные формы.
- Сохранять данные локально, в Google Sheets или в базе.
- Переводить сырые таблицы в Markdown / JSON / структурированные отчеты.
- Создавать системную память бизнеса: оферты, правила, товары, поставки, юнит-экономика, ограничения.
- Давать AI-агенту постоянный контекст через файлы, папки, базу, инструкции и `AGENTS.md`.
- Строить повторяемые циклы проверки: ежедневно, еженедельно, перед поставкой, после рекламной кампании.
- Писать конкретные скилы / инструкции для повторяемых задач.
- Подключать Google Sheets как рабочий интерфейс.
- Подключать базу данных, если таблиц становится мало.
- Работать с карточками товаров, изображениями, описаниями и отзывами.
- Скачивать и структурировать информацию по товарам.
- Проектировать MCP-серверы или плагины для интеграции с Wildberries / Ozon / внутренними сервисами.
- Запускать все это локально на компьютере или в облачной среде.
- Понимать, где нужен API, где достаточно файла, а где нужен разработчик.

## Главная схема

```text
Источники данных
  -> сбор
  -> хранилище
  -> нормализация
  -> контекст для AI
  -> анализ / проверка
  -> действие
  -> запись результата
  -> следующий цикл
```

## Слои системы

### 1. Слой данных

Откуда берем данные:

- API Wildberries / Ozon / Яндекс Маркета;
- личный кабинет и выгрузки;
- CSV / Excel;
- Google Sheets;
- ручной ввод менеджера;
- CRM;
- рекламные кабинеты;
- карточки товаров;
- отзывы и вопросы;
- изображения;
- внутренние документы.

### 2. Слой хранения

Где храним:

- папка проекта с файлами;
- Markdown-файлы;
- CSV / JSON;
- Google Sheets;
- SQLite;
- PostgreSQL;
- облачная база;
- Notion / Airtable, если нужно быстро.

Для первого потока лучше начинать с простого:

```text
папка проекта + Markdown + CSV/JSON + Google Sheets
```

Базу данных вводить как следующий уровень, когда данных много или нужны регулярные запросы.

### 3. Слой памяти бизнеса

Что сохраняем как постоянный контекст:

- описание бизнеса;
- список товаров;
- категории;
- себестоимость;
- правила расчета маржи;
- оферты;
- условия поставок;
- ограничения маркетплейса;
- правила бренда;
- требования к карточкам;
- шаблоны отчетов;
- регламенты менеджеров;
- история важных решений.

Пример файлов:

```text
business-context.md
products.md
unit-economics-rules.md
brand-rules.md
marketplace-rules.md
supplies.md
offers.md
weekly-checks.md
```

### 4. Слой нормализации

Задача:

Превратить сырые данные в удобный для AI и человека формат.

Примеры:

- из CSV сделать Markdown-отчет;
- из API-ответа сделать таблицу;
- из карточки товара сделать структурированное описание;
- из отзывов сделать список болей и возражений;
- из рекламного отчета сделать сводку по кампаниям;
- из поставок сделать календарь и риски.

### 5. Слой агентных задач

Здесь появляются повторяемые задачи:

- проверить маржинальность;
- найти проблемные SKU;
- проверить остатки;
- проверить карточки;
- подготовить ТЗ на визуал;
- найти расхождения в данных;
- подготовить отчет собственнику;
- обновить Google Sheet;
- записать выводы в Markdown.

### 6. Слой автоматизации

Чем соединяем:

- Codex / Claude Code;
- локальные скрипты;
- Google Sheets API;
- маркетплейс API;
- MCP-серверы;
- Telegram-боты;
- cron / планировщик;
- GitHub Actions или другой облачный запуск;
- плагины и скилы.

## Правильная ценность курса

Не:

```text
Мы научим вас анализировать отчет.
```

А:

```text
Мы покажем, как собрать свою систему, которая получает данные, хранит контекст бизнеса, приводит данные к понятному виду и позволяет AI-агенту регулярно проверять то, что важно именно вам.
```

## Роль 4 воркшопов

1. **Данные и хранилище**  
   Учим получать данные из API / файлов / Google Sheets и складывать их в понятную структуру.

2. **Память бизнеса и нормализация**  
   Учим создавать системный контекст: товары, правила, оферты, поставки, юнит-экономика, Markdown-слой.

3. **Циклы проверки и скилы**  
   Учим превращать повторяемые задачи в инструкции, скилы и регулярные проверки.

4. **Интеграции, карточки, визуал, MCP и плагины**  
   Учим подключать внешние источники, работать с карточками и изображениями, проектировать MCP/плагины для маркетплейсов.

## Что важно не обещать

Не обещать:

- полное управление кабинетом маркетплейса за 4 занятия;
- автоматическое принятие финансовых решений;
- обход правил маркетплейсов;
- парсинг там, где это нарушает правила площадки;
- безопасную работу с реальными ключами без настройки доступа.

Обещать:

- методологию построения AI-инструментария;
- первые рабочие связки;
- понятную архитектуру;
- шаблоны файлов и инструкций;
- прототипы интеграций;
- понимание, как развивать систему дальше.
