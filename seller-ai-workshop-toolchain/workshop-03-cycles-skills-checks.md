# Воркшоп 3: циклы проверки, скилы и регулярные AI-процессы

**Навигация:** [← README](../README.md) | [← Воркшоп 2](workshop-02-memory-normalization.md) | [Далее: воркшоп 4](workshop-04-integrations-mcp-plugins-visuals.md)

## Суть

Этот воркшоп про то, как превратить разовые запросы к AI в повторяемые процессы: ежедневные, еженедельные и событийные проверки.

Главный результат:

```text
У участника появляется 1-2 регулярных AI-процесса и черновик собственного скила / инструкции для агента.
```

## Что показываем

### 1. Что такое цикл проверки

Цикл:

```text
Получить данные -> прочитать контекст -> проверить по правилам -> записать выводы -> предложить действия -> сохранить результат
```

Примеры циклов:

- ежедневная проверка остатков;
- еженедельный отчет по марже;
- проверка товаров перед поставкой;
- проверка рекламных расходов;
- проверка карточек перед запуском рекламы;
- анализ отзывов раз в неделю;
- сводка задач менеджера.

### 2. Чем цикл отличается от разового промпта

Разовый промпт:

```text
Проанализируй продажи.
```

Цикл:

```text
Каждый понедельник:
1. взять последний sales_processed.csv;
2. прочитать business-context.md и unit-economics-rules.md;
3. проверить SKU по правилам;
4. записать findings в reports/weekly;
5. обновить Google Sheet;
6. сформировать действия для собственника.
```

### 3. Файл регулярной проверки

Создаем:

```text
checks/weekly-margin-check.md
```

Шаблон:

```text
# Еженедельная проверка маржи

## Цель

## Какие файлы читать

## Какие правила применять

## Что проверить

## Формат результата

## Что записать в отчет

## Что считать проблемой

## Что не делать
```

### 4. Скил / инструкция для агента

Скил - это повторяемая инструкция для конкретного типа задачи. Он нужен, чтобы каждый раз не объяснять агенту одно и то же.

Примеры скилов:

- `wb-margin-check`;
- `product-card-review`;
- `review-analysis`;
- `supply-risk-check`;
- `weekly-owner-report`;
- `ad-spend-check`.

Пример структуры:

```text
skills/
  weekly-owner-report/
    SKILL.md
```

## Что делают участники

1. Выбирают одну регулярную проверку.
2. Описывают цикл.
3. Создают файл проверки в `checks/`.
4. Создают черновик скила в `skills/`.
5. Прогоняют тест на своих или демо-данных.
6. Сохраняют результат в `reports/`.

## Практические циклы, которые можно выбрать

- еженедельная маржа;
- карточки для улучшения;
- поставки и остатки;
- отзывы и вопросы;
- рекламные расходы;
- ежедневная сводка менеджера.

## Артефакты

- папка `checks/`;
- файл первой регулярной проверки;
- папка `skills/`;
- черновик `SKILL.md`;
- первый отчет в `reports/`;
- понимание, как запускать цикл вручную и как потом автоматизировать.

## Домашнее задание

1. Доработать один скил.
2. Запустить проверку еще раз на новых данных.
3. Сравнить два результата.
4. Описать, что должно запускаться автоматически.
5. Подготовить интеграцию или визуальный процесс для следующего воркшопа.
