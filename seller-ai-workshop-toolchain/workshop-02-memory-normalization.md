# Воркшоп 2: системная память бизнеса и нормализация данных

**Навигация:** [← README](../README.md) | [← Воркшоп 1](workshop-01-data-api-storage.md) | [Далее: воркшоп 3](workshop-03-cycles-skills-checks.md)

## Суть

Этот воркшоп про то, как создать постоянную память бизнеса для AI-агента и перевести сырые данные в формат, который удобно анализировать.

Главный результат:

```text
У участника появляется контекстная база бизнеса: товары, правила, оферты, поставки, юнит-экономика, структура Markdown/JSON-файлов.
```

## Что показываем

### 1. Что такое системная память

AI-агент не должен каждый раз заново спрашивать:

- какие товары продаем;
- как считать маржу;
- какие комиссии учитывать;
- какие категории важны;
- какие оферты и условия действуют;
- какие поставки ожидаются;
- какие правила бренда соблюдать.

Все это нужно хранить в файлах или базе.

### 2. Папка `context/`

Пример:

```text
context/
  business-context.md
  products.md
  product-costs.md
  marketplace-rules.md
  unit-economics-rules.md
  offers.md
  supplies.md
  brand-rules.md
  report-rules.md
```

### 3. Нормализация сырых данных

Показываем цепочку:

```text
data/raw/sales.csv
  -> data/processed/sales-normalized.csv
  -> reports/weekly/sales-summary.md
```

Задача агенту:

```text
Возьми сырой файл продаж.
Не делай финальный анализ.
Сначала приведи данные к нормальной структуре:
- понятные названия столбцов;
- единый формат дат;
- числовые поля;
- отдельные поля для SKU, названия, выручки, расходов;
- список проблем в данных.

Сохрани описание структуры в Markdown.
```

### 4. Markdown как слой между таблицей и мышлением

Markdown удобен, потому что:

- его читает человек;
- его читает агент;
- его легко версионировать;
- в нем можно хранить выводы;
- он подходит для отчетов и инструкций.

Пример `sales-summary.md`:

```text
# Сводка продаж

Период:
Источник:
Количество строк:
Поля:
Проблемы данных:
Что можно анализировать:
Что нельзя анализировать без дополнительных данных:
```

## Что делают участники

1. Создают папку `context/`.
2. Заполняют `business-context.md`.
3. Создают `products.md`.
4. Создают `unit-economics-rules.md`.
5. Создают `offers.md` или `supplies.md`, если это важно для бизнеса.
6. Берут один сырой файл и превращают его в Markdown-описание.
7. Формируют правило: какие данные агент должен читать перед анализом.

## Артефакты

- `business-context.md`;
- `products.md`;
- `unit-economics-rules.md`;
- `offers.md` / `supplies.md`;
- первый Markdown-отчет о структуре данных;
- понимание, где хранить постоянные знания бизнеса.

## Домашнее задание

1. Добавить 5-20 ключевых товаров в `products.md`.
2. Записать правила расчета маржи.
3. Описать актуальные оферты / поставки.
4. Прогнать один сырой отчет через нормализацию.
5. Подготовить одну регулярную проверку для следующего воркшопа.
