# Урок 1: базовый ввод в Codex / Claude Code для селлера

**Навигация:** [← README](README.md) | [← План занятий](ai-agents-seller-workshop-lessons-plan.md) | [Далее: инструментальная методология](seller-ai-workshop-toolchain/00-toolchain-methodology.md)

## Роль первого урока

Первый урок должен снять страх перед AI-агентами и дать участнику ясную картину:

AI-агент - это не "страшное программирование", а рабочий помощник, которому можно поручать задачи с файлами, таблицами, текстами, карточками, отчетами, кодом и автоматизацией. Главный акцент курса: не разовый анализ, а сборка системы, которая получает данные, хранит контекст бизнеса и запускает повторяемые проверки.

Главная задача урока - не научить всех программировать. Главная задача - показать, как селлеру начать использовать Codex / Claude Code как рабочий инструмент.

## Что обязательно нужно рассказать

Да, в первом уроке нужно рассказать:

1. Что такое Codex / Claude Code простым языком.
2. Чем AI-агент отличается от обычного ChatGPT.
3. Зачем нужен VS Code или другая рабочая среда.
4. Как в целом устанавливается и запускается Codex.
5. Какие есть варианты работы: IDE, CLI, облако, чат.
6. Что AI-агент может делать для селлера.
7. Какие ограничения и правила безопасности есть.
8. Как правильно дать первую задачу.
9. Как выглядит путь "сырые данные -> память бизнеса -> проверка -> отчет".
10. Почему нейросети - это не страшно, а рабочий инструмент.

Но важно: установку не надо превращать в половину урока. Установка - это подготовка рабочего места. Основная ценность урока - показать, что с этим делать.

## Цель урока

После урока участник должен:

- понимать, что такое AI-агент;
- понимать, зачем селлеру Codex / Claude Code;
- знать, какие рабочие задачи можно решать;
- понимать базовую логику установки и запуска;
- уметь сформулировать первую простую задачу;
- понимать базовую архитектуру будущей AI-системы селлера;
- увидеть 2-3 демонстрации на задачах селлера;
- перестать воспринимать нейросети как "игрушку для текстов" или "сложное программирование".

## Формат

Рекомендуемая длительность: 90-120 минут.

Если это бесплатное вводное занятие, лучше 90 минут.

Если это первый урок платного блока, лучше 120 минут, чтобы участники успели установить инструменты и сделать первую практику.

## Главный тезис урока

```text
Codex / Claude Code - это не замена селлера и не магическая кнопка.
Это рабочий помощник, которому можно поручать понятные задачи, если дать ему контекст, данные и критерии проверки.
```

## Структура урока на 90 минут

### Блок 1: Снятие страха и объяснение рамки, 10 минут

Что сказать:

Многие думают, что Codex и Claude Code - это только для программистов. На самом деле селлеру не обязательно становиться разработчиком. Важно другое: научиться описывать свои процессы так, чтобы AI-агент мог помочь с анализом, таблицами, карточками, отчетами, текстами, ТЗ и автоматизацией.

Что донести:

- нейросеть не нужно "бояться";
- это не магия и не кнопка "сделай бизнес";
- это рабочий инструмент, как Excel, CRM, сервис аналитики или рекламный кабинет;
- результат нужно проверять;
- чем точнее задача, тем полезнее ответ.

Пример простой аналогии:

```text
Обычный ChatGPT похож на консультанта, которому можно задать вопрос.
Codex / Claude Code ближе к помощнику, который может открыть папку, посмотреть файлы, внести изменения, собрать шаблон, подготовить структуру и показать, что именно сделал.
```

### Блок 2: Что такое Codex и Claude Code, 10 минут

Объяснение простым языком:

Codex и Claude Code - это AI-агенты для работы с задачами, где есть файлы, структура, логика, инструкции и результат. Они могут помогать писать код, но для селлера важнее другое: они могут помогать собирать рабочие инструменты и разбирать данные.

Что может агент:

- читать файлы;
- анализировать таблицы;
- помогать с отчетами;
- создавать Markdown-документы;
- писать инструкции и ТЗ;
- собирать прототипы страниц, ботов, скриптов и мини-сервисов;
- предлагать структуру автоматизации;
- запускать команды в рабочей среде, если это разрешено;
- править файлы и показывать изменения.

Что важно объяснить:

- агент не "знает ваш бизнес" сам по себе;
- ему нужно дать контекст;
- он может ошибаться;
- цифры и бизнес-решения нужно проверять;
- доступы, токены, пароли и персональные данные нельзя передавать бездумно.

### Блок 3: Чем это отличается от ChatGPT, 10 минут

Сравнение:

| Обычный ChatGPT | Codex / Claude Code |
| --- | --- |
| Отвечает в чате | Работает с проектом, файлами и задачами |
| Хорош для идей и текстов | Хорош для создания рабочих артефактов |
| Обычно не видит вашу папку | Может работать с файлами в рабочей среде |
| Дает совет | Может сделать черновик решения |
| Нужно вручную переносить результат | Может создавать и править файлы |

Главная мысль:

ChatGPT отвечает. AI-агент помогает делать.

Для селлера это значит:

- можно не просто спросить "как посчитать маржу", а дать таблицу и попросить собрать шаблон расчета;
- можно не просто спросить "как улучшить карточку", а попросить сделать ТЗ, структуру слайдов и список гипотез;
- можно не просто спросить "как сделать бота", а получить сценарий, структуру данных и прототип.

### Блок 4: Рабочее место: VS Code, Codex, Claude Code, 15 минут

Цель блока:

Показать участникам, что им нужно подготовить для практики.

Важно:

Не надо делать из этого сложный технический туториал. Нужно объяснить, что рабочая среда - это место, где лежат файлы проекта и где AI-агент может с ними работать.

Что такое VS Code:

VS Code - это редактор, в котором удобно держать файлы, таблицы, инструкции, шаблоны, код, документы и подключать AI-агентов через расширения.

Что нужно подготовить:

1. Установить VS Code.
2. Установить расширение Codex для VS Code или использовать Codex CLI.
3. Войти через ChatGPT-аккаунт или API-ключ, если выбран такой способ.
4. Создать папку проекта, например `seller-ai-workshop`.
5. Положить туда первые файлы: описание бизнеса, пример отчета, карточку товара, список задач.

Что сказать про актуальность:

```text
Перед занятием мы дадим отдельную инструкцию по установке. Интерфейсы и команды могут обновляться, поэтому установочный чек-лист нужно сверять с официальной документацией OpenAI / Anthropic перед запуском потока.
```

Минимальный вариант для новичка:

- поставить VS Code;
- установить Codex IDE extension;
- войти в аккаунт;
- открыть папку проекта;
- написать первую задачу в боковой панели Codex.

Альтернативный вариант:

- использовать Codex CLI в терминале;
- запустить команду `codex`;
- работать в интерактивном режиме.

Не стоит требовать CLI от всех новичков на первом бесплатном уроке. Для селлеров проще начинать с IDE или демонстрации экрана преподавателя.

### Блок 5: Безопасность и границы, 10 минут

Что объяснить:

AI-агент может быть очень полезен, но ему нельзя бездумно отдавать все доступы.

Нельзя передавать без необходимости:

- пароли;
- API-ключи;
- токены кабинетов;
- персональные данные клиентов;
- полные финансовые выгрузки без обезличивания;
- доступы к рекламным кабинетам;
- данные, которые нельзя показывать подрядчикам.

Что можно давать на старте:

- обезличенные таблицы;
- фрагменты отчетов;
- примеры карточек;
- скриншоты без чувствительных данных;
- описание процесса словами;
- тестовые данные;
- копии файлов без доступов и секретов.

Главная мысль:

```text
AI-агенту нужно давать достаточно контекста для задачи, но не больше доступа, чем требуется.
```

### Блок 6: Возможности для селлера, 15 минут

Показать карту применений.

Аналитика:

- привести отчет продаж к нормальной структуре;
- найти проблемные SKU;
- посчитать маржинальность;
- выделить товары с риском остатка;
- подготовить еженедельный отчет.

Финансы:

- собрать шаблон юнит-экономики;
- проверить рекламные расходы;
- найти товары с отрицательной прибылью;
- посчитать влияние скидок и логистики.

Карточки и визуал:

- собрать профиль карточки товара;
- сформулировать УТП;
- подготовить ТЗ дизайнеру;
- вытащить боли из отзывов;
- придумать гипотезы для первого слайда.

Операционка:

- сделать чек-листы;
- описать процессы;
- подготовить регламенты для менеджера;
- собрать ежедневный отчет.

Автоматизация:

- спроектировать Telegram-бота;
- описать мини-сервис;
- подготовить структуру базы или таблицы;
- сделать прототип внутреннего кабинета;
- автоматизировать обработку CSV / Excel.

Ключевая мысль:

AI-агент особенно полезен там, где есть повторяемая задача и понятный формат результата.

### Блок 7: Демонстрация 1. Первая задача для агента, 10 минут

Цель:

Показать самый простой старт без сложной установки.

Демо:

Создаем файл `seller-context.md`:

```text
Я селлер на WB / Ozon.
Продаю товары в категории ...
Главные задачи: анализ маржи, улучшение карточек, контроль остатков, реклама.
Хочу использовать AI-агента как помощника.
```

Задача агенту:

```text
Прочитай файл seller-context.md.
Предложи 10 задач, где AI-агент может помочь моему селлерскому бизнесу.
Раздели их по категориям: аналитика, финансы, карточки, операционка, автоматизация.
Для каждой задачи напиши, какие данные нужны и какой результат можно получить.
```

Что показать:

- агент читает контекст;
- предлагает задачи;
- структурирует их;
- можно попросить уточнить и выбрать первые 3 задачи.

### Блок 8: Демонстрация 2. Путь от сырого файла к AI-проверке, 10 минут

Цель:

Показать не финальный анализ отчета, а правильный путь работы с данными.

Демо:

```text
data/raw/sales-example.csv
  -> описание структуры данных
  -> data/processed/sales-normalized.csv
  -> reports/sales-summary.md
```

Задача агенту:


```text
Посмотри файл data/raw/sales-example.csv.
Не делай финальный бизнес-анализ.
Сначала опиши структуру данных: какие есть столбцы, что они означают, какие поля нужны для дальнейшей проверки маржи и остатков.
Предложи, как привести файл к нормальной структуре, и создай Markdown-сводку для дальнейшей работы.
```

Что показать:

- агент сначала понимает структуру данных;
- отделяем сырые данные от обработанных;
- сохраняем выводы в Markdown;
- анализ начинается только после того, как данные и правила понятны.

Дополнительный вариант, если нужно показать карточки:

Дать описание товара и попросить:

```text
Разбери карточку товара.
Предложи структуру первого слайда, 5 идей для инфографики и ТЗ дизайнеру.
```

Лучше выбрать один вариант для первого урока, чтобы не перегрузить участников.

Рекомендация:

Для бесплатного урока показать путь от сырого файла к Markdown-сводке, потому что он сразу объясняет методологию курса: сначала данные и структура, потом проверки и выводы.

### Блок 9: Практика участников, 10 минут

Задание:

Каждый участник пишет одну реальную задачу по шаблону.

Шаблон:

```text
Я хочу, чтобы AI-агент помог мне с ...
Сейчас я делаю это так ...
У меня есть такие данные ...
На выходе я хочу получить ...
Проверять результат я буду по ...
```

Примеры:

- "Хочу понять, какие товары убыточны".
- "Хочу подготовить ТЗ дизайнеру для новой карточки".
- "Хочу сделать ежедневный отчет менеджера".
- "Хочу собрать Telegram-бота для заявок".
- "Хочу автоматически разбирать рекламные расходы".

### Блок 10: Итог и следующий шаг, 10 минут

Что зафиксировать:

- AI-агенты - это рабочий инструмент, а не страшная технология;
- селлеру важно научиться ставить задачи;
- начинать нужно с простых процессов;
- данные нужно давать аккуратно;
- результат нужно проверять;
- лучший первый шаг - выбрать одну задачу и разобрать ее с агентом.

Переход к следующим занятиям:

```text
На первом уроке мы разобрались, что такое AI-агенты и как с ними начать.
Дальше будем работать практично: возьмем реальные задачи селлеров и разложим их на аналитику, карточки, финансы и автоматизацию.
```

## Структура урока на 120 минут

Если урок проводится как первый платный урок, можно добавить 30 минут на установку и настройку.

Дополнительный блок: установка и проверка рабочего места, 30 минут.

Что сделать:

1. Проверить, что у участника установлен VS Code.
2. Установить или открыть Codex IDE extension.
3. Войти в аккаунт.
4. Создать папку проекта.
5. Создать первый файл `seller-context.md`.
6. Запустить первую задачу.
7. Убедиться, что участник понял, где писать запрос, где смотреть файлы и как принимать изменения.

Важно:

Если группа большая и уровень разный, установку лучше вынести в отдельную инструкцию до урока, а на уроке оставить 10-15 минут на проверку.

## Что подготовить преподавателю

Файлы:

- `seller-context.md` - пример описания бизнеса селлера;
- `sales-example.csv` - обезличенная таблица продаж;
- `product-card-example.md` - описание товара и текущей карточки;
- `first-prompts.md` - набор первых промптов;
- `security-checklist.md` - чек-лист, что нельзя передавать агенту.

Демо:

1. Открыть папку проекта в VS Code.
2. Показать, где находится Codex.
3. Показать первый запрос.
4. Показать, как агент читает файл.
5. Показать, как агент создает новый Markdown-файл с результатом.
6. Показать, как попросить улучшить результат.

## Первый набор промптов

### Промпт 1: карта задач

```text
Прочитай описание моего селлерского бизнеса.
Предложи 10 задач, где AI-агент может помочь.
Раздели задачи на категории: аналитика, финансы, карточки, операционка, автоматизация.
Для каждой задачи укажи, какие данные нужны и какой результат получится.
```

### Промпт 2: превращение боли в ТЗ

```text
Помоги превратить мою бизнес-боль в понятную задачу для AI-агента.

Боль:
...

Задай мне уточняющие вопросы, а потом составь ТЗ по структуре:
цель, контекст, входные данные, действия, формат результата, критерии проверки.
```

### Промпт 3: анализ таблицы

```text
Проанализируй таблицу как помощник селлера маркетплейса.
Найди проблемные товары, товары с низкой маржинальностью, товары с риском остатка и товары, которые стоит проверить в рекламе.
Сделай выводы простым языком и предложи действия на неделю.
```

### Промпт 4: карточка товара

```text
Разбери карточку товара как маркетолог маркетплейса.
Найди слабые места оффера, первого слайда и инфографики.
Предложи новую структуру первого слайда, 5 идей для инфографики и ТЗ дизайнеру.
```

### Промпт 5: автоматизация процесса

```text
Помоги спроектировать автоматизацию для моего процесса.

Сейчас процесс работает так:
...

Мне нужно:
...

Сделай схему процесса, сценарий Telegram-бота или мини-сервиса, структуру данных и план первой версии.
```

## Что не нужно делать на первом уроке

- Не уходить глубоко в программирование.
- Не объяснять всю архитектуру AI-агентов.
- Не сравнивать десятки моделей.
- Не грузить участников сложными настройками.
- Не обещать, что агент сам будет управлять кабинетом маркетплейса.
- Не просить участников сразу подключать реальные API-ключи.
- Не заставлять всех работать через терминал, если аудитория не техническая.

## Что нужно сделать на первом уроке обязательно

- Объяснить простым языком, что такое AI-агент.
- Показать, чем он отличается от ChatGPT.
- Показать рабочее место.
- Показать первую задачу.
- Показать 1 практическую демонстрацию на задаче селлера.
- Дать участникам шаблон постановки задачи.
- Собрать реальные задачи участников.
- Завершить понятным следующим шагом.

## Результат первого урока

Участник должен уйти с пониманием:

- AI-агент - это рабочий инструмент;
- начинать можно без глубоких технических знаний;
- VS Code / Codex - это рабочая среда, а не "программирование ради программирования";
- агенту нужно давать контекст и данные;
- результат нужно проверять;
- первые задачи для селлера: источники данных, контекст бизнеса, нормализация, проверки, карточки, отчеты, процессы, автоматизация.

Артефакт после урока:

- файл `seller-context.md`;
- список 10 задач для AI-агента;
- одна задача, оформленная как ТЗ;
- понимание, какие источники данных и файлы нужны для первого воркшопа;
- понимание, что делать на следующем занятии.

## Ценность после первого занятия

После первого занятия участник должен получить не просто "знакомство с Codex", а конкретную пользу для себя и своего бизнеса.

### 1. Понимание, что AI-агенты - это рабочий инструмент

До урока участник может думать:

- "это для программистов";
- "я не разберусь";
- "нейросети только тексты пишут";
- "мне нужен разработчик, чтобы что-то автоматизировать";
- "это сложно и страшно".

После урока он должен понять:

- AI-агент может помогать с обычными задачами селлера;
- начинать можно с простых файлов, таблиц и описаний процессов;
- не нужно сразу писать код;
- важно не "знать программирование", а уметь описывать задачу;
- результат можно проверять и улучшать пошагово.

### 2. Понимание разницы между ChatGPT и Codex / Claude Code

Ценность:

Участник понимает, почему это не просто еще один чат.

Что должно стать ясно:

- ChatGPT чаще помогает ответом, идеей или текстом;
- Codex / Claude Code могут работать с файлами, проектами, таблицами, структурой и задачами;
- агент может не только советовать, но и создавать артефакты: Markdown-документы, шаблоны, ТЗ, прототипы, скрипты, структуру бота;
- агент полезен там, где есть повторяемая задача и понятный результат.

### 3. Участник видит свои первые задачи для AI

Ценность:

Он перестает думать абстрактно "надо как-то использовать AI" и видит список конкретных применений в своем бизнесе.

После занятия у него должен быть список задач по категориям:

- аналитика;
- финансы;
- карточки;
- визуал;
- реклама;
- операционка;
- автоматизация;
- Telegram-боты;
- внутренние отчеты.

Примеры осознаний:

- "Я могу начать с одного сырого файла и привести его к нормальной структуре".
- "Я могу создать память бизнеса: товары, правила маржи, оферты, поставки".
- "Я могу описать регулярную проверку, которую агент будет запускать по правилам".
- "Я могу подключить Google Sheets как первый интерфейс".
- "Я могу начать с одной связки данных, а не с большой разработки".

### 4. Участник учится формулировать задачу

Ценность:

Это главный навык первого урока. Не "написать красивый промпт", а превратить бизнес-боль в рабочую задачу.

До:

```text
Помоги с аналитикой.
```

После:

```text
Проанализируй таблицу продаж как помощник селлера.
Найди товары с низкой маржинальностью, высоким рекламным расходом и риском остатка.
Сделай выводы простым языком и предложи действия на неделю.
```

Что должен освоить участник:

- указать роль агента;
- дать контекст бизнеса;
- описать входные данные;
- сформулировать результат;
- задать формат ответа;
- указать критерии проверки.

### 5. Участник понимает, какие данные нужны для работы

Ценность:

Селлер начинает видеть свои отчеты, таблицы, карточки и процессы как материал для AI-агента.

Он понимает, что можно подготовить:

- выгрузку продаж;
- отчет по рекламе;
- список SKU;
- остатки;
- описание товара;
- отзывы;
- скрины карточек;
- описание ручного процесса;
- регламент менеджера;
- пример ежедневного отчета.

Это переводит AI из абстракции в рабочий процесс.

### 6. Участник узнает базовые правила безопасности

Ценность:

Он понимает, что AI можно использовать безопасно и аккуратно.

Что должно закрепиться:

- не передавать пароли и токены;
- не подключать реальные API на первом шаге;
- обезличивать данные;
- начинать с копий файлов;
- проверять цифры;
- не давать агенту больше доступа, чем нужно для задачи.

### 7. Участник получает первый артефакт

Ценность:

После урока у него должно остаться что-то конкретное, а не только впечатление.

Минимальный набор:

- `seller-context.md` - описание бизнеса;
- список 10 задач для AI-агента;
- одна задача, оформленная как ТЗ;
- список источников данных;
- понимание, какие данные подготовить дальше.

Лучший результат:

Участник уже видит одну связку данных или процесс, который можно взять на следующий платный блок и превратить в часть собственной AI-системы.

### 8. Участник понимает следующий шаг

Ценность:

После первого занятия не должно быть ощущения "интересно, но непонятно что делать".

Следующий шаг должен быть простым:

1. Выбрать одну задачу.
2. Подготовить данные.
3. Описать ее по шаблону.
4. На следующем занятии разобрать ее с агентом.

Формулировка для участника:

```text
Ваша задача после первого урока - не автоматизировать весь бизнес.
Ваша задача - выбрать один источник данных или один повторяющийся процесс, подготовить пример данных и принести его на сборку первого слоя системы.
```

## Короткая формула ценности первого занятия

```text
После первого занятия селлер понимает, что AI-агент может быть рабочим помощником, видит первые источники данных и процессы для автоматизации, умеет описать одну задачу по шаблону и знает, какие данные подготовить для сборки своей AI-системы.
```
