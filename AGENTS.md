## Контекст проекта

Проект: воркшоп для селлеров маркетплейсов по использованию Codex / Claude Code и AI-агентов в рабочих задачах.

Целевая аудитория:

- селлеры WB / Ozon / Яндекс Маркета;
- собственники селлерского бизнеса;
- менеджеры маркетплейсов;
- аналитики, контент-менеджеры, эксперты и агентства, работающие с селлерами.

Главная идея:

AI-агенты нужно объяснять не как "программирование", а как новый рабочий инструмент селлера для аналитики, финансов, карточек, визуала, автоматизации, Telegram-ботов и внутренних процессов.

## Рабочие файлы

- `ai-agents-seller-workshop-promo-and-next-steps.md` - рекламные тексты и план запуска.
- `ai-agents-seller-workshop-lessons-plan.md` - общий план бесплатного занятия и 4 инструментальных воркшопов.

- `seller-ai-workshop-toolchain/` - инструментальная версия воркшопов: данные, API, память, нормализация, циклы, скилы, MCP, плагины и интеграции.

## Правила подготовки материалов

- Все новые крупные блоки сохранять в отдельные Markdown-файлы, если пользователь просит "отдельно" или материал становится самостоятельным.
- Каждый новый Markdown-файл должен иметь навигацию: сверху ссылку на `README.md`, а снизу блок "Назад / Далее / README".
- Писать на русском языке.
- Сохранять практический тон: меньше абстрактной теории, больше сценариев, задач, демонстраций и результатов.
- Не обещать, что участник полностью автоматизирует бизнес за 1-4 занятия.
- Обещать реалистичный результат: участник поймет принцип работы AI-агентов, научится строить систему данных, памяти, проверок и интеграций, соберет первые инструментальные заготовки.
- Для установочных инструкций Codex, OpenAI API, моделей и актуальных возможностей сверяться с официальными источниками перед финальной публикацией.
- Объяснять Codex / Claude Code простым языком: "помощник, который может работать с файлами, таблицами, кодом, командами и проектами", а не "сложная среда разработки".
- В материалах для селлеров избегать перегруза техническими терминами. Если термин нужен, сразу объяснять его через рабочий пример.

## Методология курса

Базовая формула работы:

```text
Бизнес-боль -> данные -> задача для агента -> черновик решения -> проверка -> улучшение -> внедрение
```

Для инструментальной версии курса использовать более точную формулу:

```text
Источники данных -> сбор -> хранилище -> нормализация -> системная память -> AI-проверка -> отчет / действие -> следующий цикл
```

Если пользователь говорит про API, MCP, базу, плагины, скилы, Google Sheets, карточки товаров или автоматизацию, фокус должен быть не на ручном анализе отчета, а на построении инструментария, который позволит селлеру самому собирать и проверять данные.

Каждый урок или воркшоп должен отвечать на вопросы:

1. Какую боль селлера закрываем?
2. Какие данные или материалы нужны?
3. Что участник делает руками?
4. Что делает AI-агент?
5. Как проверить результат?
6. Какой артефакт остается после занятия?

## Стиль уроков

Хороший урок строится так:

1. Простое объяснение.
2. Демонстрация на понятном примере селлера.
3. Практика участника.
4. Шаблон, который можно забрать.
5. Домашнее задание или следующий шаг.

Нельзя строить урок как обзор функций ради функций. Каждая функция Codex / Claude Code должна быть привязана к бизнес-задаче селлера.
