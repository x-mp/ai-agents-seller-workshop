# AI-инструменты для кабинета продавца на Wildberries и Ozon

**Практический воркшоп по Codex / Claude Code: данные из кабинета, API, Google Sheets, карточки, остатки, поставки, реклама и автоматические проверки.**

**Навигация:** [Общий план](ai-agents-seller-workshop-plan.md) | [Первый урок](ai-agents-seller-workshop-lesson-01.md) | [Методология](seller-ai-workshop-toolchain/00-toolchain-methodology.md) | [Воркшоп 1](seller-ai-workshop-toolchain/workshop-01-data-api-storage.md) | [Воркшоп 2](seller-ai-workshop-toolchain/workshop-02-memory-normalization.md) | [Воркшоп 3](seller-ai-workshop-toolchain/workshop-03-cycles-skills-checks.md) | [Воркшоп 4](seller-ai-workshop-toolchain/workshop-04-integrations-mcp-plugins-visuals.md)

## О чем этот курс

У продавца на Wildberries или Ozon каждый день появляются одни и те же вопросы:

- что происходит с продажами;
- какие товары просели;
- где заканчиваются остатки;
- где реклама тратит деньги без результата;
- какие карточки надо переделать;
- какие поставки нужно проверить;
- где изменилась маржа;
- какие отчеты снова надо выгрузить, свести и объяснить команде.

Обычно все это живет в разных местах: в личном кабинете, Excel, Google Sheets, рекламном кабинете, чатах с менеджерами, файлах поставок, карточках товаров и голове собственника.

Главная идея курса: показать, как с помощью Codex / Claude Code начать собирать из этого не хаос, а рабочую систему.

Не просто спросить у нейросети: "что делать с продажами?"

А собрать понятный процесс:

```text
кабинет WB/Ozon -> выгрузка или API -> таблица/файл -> нормализация -> правила продавца -> AI-проверка -> отчет / задача / уведомление
```

## Что мы не делаем

Мы не продаем обещание "за 4 занятия полностью автоматизируем кабинет".

Мы не делаем вид, что AI сам будет управлять рекламой, поставками и ценами без контроля.

Мы не строим абстрактный курс по нейросетям.

## Что мы делаем

Мы учим продавца и его команду собирать инструменты вокруг реальных задач кабинета:

- как забрать данные из кабинета WB/Ozon или из выгрузки;
- как хранить сырые и обработанные отчеты;
- как подключить Google Sheets как понятный рабочий интерфейс;
- как описать товары, себестоимость, оферты, поставки и правила маржи так, чтобы AI видел контекст;
- как превращать сырые таблицы в понятный Markdown / JSON / обработанные таблицы;
- как запускать регулярные проверки по своим правилам;
- как готовить данные по карточкам, изображениям, отзывам и конкурентам;
- как проектировать Telegram-уведомления, MCP-инструменты, плагины и будущие внутренние сервисы.

## Почему это ценно

Codex / Claude Code полезны не потому, что "умеют писать код".

Для продавца важнее другое: они могут работать с файлами, папками, таблицами, инструкциями, правилами и повторяемыми задачами.

Если правильно подготовить данные и контекст, AI-агент может помогать:

- разложить выгрузку по понятным полям;
- объяснить, каких данных не хватает для проверки маржи;
- подготовить структуру Google Sheets;
- собрать файл с правилами расчета;
- сделать регулярную проверку остатков;
- сформировать список товаров, которые надо проверить руками;
- подготовить профиль карточки товара;
- написать ТЗ для разработчика, бота, MCP-инструмента или внутреннего сервиса.

То есть AI становится не "чатом для советов", а рабочим помощником внутри процессов продавца.

## Что меняется у участника

До курса:

```text
Есть личный кабинет, выгрузки, таблицы, реклама, карточки, остатки и много ручной проверки.
AI вроде полезен, но непонятно, как применить его к своим данным.
```

После курса:

```text
Есть понятная схема: какие данные брать из WB/Ozon, где их хранить,
как давать AI правила и контекст, какие проверки запускать регулярно
и куда отдавать результат: отчет, Google Sheets, Telegram или задачу команде.
```

## Для кого

Курс подходит:

- собственникам, которые хотят лучше контролировать кабинет WB/Ozon;
- менеджерам маркетплейсов, которые устали от ручных отчетов;
- аналитикам, которые хотят быстрее готовить проверки и выводы;
- контент-менеджерам, которые работают с карточками, отзывами и визуалом;
- экспертам и агентствам, которые хотят упаковывать свои методики в инструменты.

Курс особенно полезен тем, у кого уже есть:

- товары и регулярные продажи;
- выгрузки из кабинета;
- Google Sheets или Excel-таблицы;
- рекламные расходы;
- поставки и остатки;
- карточки, которые нужно улучшать;
- повторяющиеся проверки, которые занимают время каждую неделю.

## Что собираем на практике

На воркшопах участник собирает не "конспект", а основу своей рабочей системы:

- карту источников данных из кабинета WB/Ozon;
- структуру проекта для AI-агента;
- папки `data/`, `context/`, `reports/`, `checks/`, `skills/`, `integrations/`;
- описание товаров, себестоимости, оферт, поставок и правил маржи;
- правила нормализации сырых CSV / Excel / API-ответов;
- Markdown-отчеты, которые читает и человек, и AI;
- регулярные проверки по остаткам, марже, рекламе, карточкам или поставкам;
- черновики скилов и инструкций для повторяемых задач;
- карту интеграций с API, Google Sheets, Telegram, MCP и будущими плагинами;
- профиль карточки товара: описание, изображения, отзывы, конкуренты, выводы.

## Программа

### Бесплатное вводное занятие

Показываем, что Codex / Claude Code - это не страшное программирование, а рабочая среда, где можно собирать файлы, инструкции, данные и проверки.

Главный результат: участник понимает, какие данные из своего кабинета и какие процессы можно первыми привести в порядок.

[Открыть план первого урока](ai-agents-seller-workshop-lesson-01.md)

### Воркшоп 1: данные, API, файлы и хранилище

Разбираем, откуда брать данные: личный кабинет WB/Ozon, API, выгрузки, CSV, Excel, Google Sheets, карточки, отзывы, ручные файлы менеджеров.

Зачем это нужно: без нормального входа данных AI будет давать красивые, но бесполезные ответы. Сначала нужно понять, какие данные есть, где они лежат и как их хранить.

[Открыть воркшоп 1](seller-ai-workshop-toolchain/workshop-01-data-api-storage.md)

### Воркшоп 2: память кабинета и нормализация

Создаем постоянный контекст: товары, себестоимость, правила маржи, поставки, оферты, ограничения, правила отчетов.

Зачем это нужно: AI не должен каждый раз заново спрашивать, как считать маржу, какие товары важны и какие правила учитывать.

[Открыть воркшоп 2](seller-ai-workshop-toolchain/workshop-02-memory-normalization.md)

### Воркшоп 3: регулярные проверки и скилы

Превращаем разовые запросы в повторяемые процессы: еженедельная проверка маржи, остатков, рекламы, карточек, отзывов или поставок.

Зачем это нужно: ценность появляется не когда AI один раз ответил, а когда проверка запускается по понятной схеме снова и снова.

[Открыть воркшоп 3](seller-ai-workshop-toolchain/workshop-03-cycles-skills-checks.md)

### Воркшоп 4: интеграции, MCP, плагины и визуальные данные

Проектируем связки с API, Google Sheets, Telegram, карточками товаров, изображениями, отзывами и будущими MCP-инструментами.

Зачем это нужно: результат должен попадать туда, где работает команда: в таблицу, отчет, Telegram, задачу менеджеру или внутренний сервис.

[Открыть воркшоп 4](seller-ai-workshop-toolchain/workshop-04-integrations-mcp-plugins-visuals.md)

## Что участник сможет делать после курса

- Понимать, какие данные нужны из кабинета WB/Ozon.
- Подготовить структуру проекта для работы с Codex / Claude Code.
- Разложить выгрузки и API-ответы по понятным папкам.
- Сохранять правила товаров, поставок и маржи как постоянный контекст.
- Подключать Google Sheets как первый рабочий интерфейс.
- Создавать регулярные AI-проверки.
- Проектировать интеграции и будущие плагины.
- Говорить с разработчиком конкретно: какие данные нужны, куда их сохранять, что проверять и какой результат отдавать.

## Главная ценность

У каждого продавца разные товары, разные категории, разные отчеты и разные правила.

Поэтому курс не дает один универсальный шаблон "для всех". Он дает подход, с помощью которого продавец может собрать свою систему под свой кабинет WB/Ozon.

Это переход от:

```text
я каждый раз руками собираю отчеты и спрашиваю AI общие вопросы
```

к:

```text
у меня есть структура данных, правила, регулярные проверки и понятный путь к автоматизации
```
