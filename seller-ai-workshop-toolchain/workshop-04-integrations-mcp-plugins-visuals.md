# Воркшоп 4: интеграции, MCP, плагины, карточки и визуальные данные

**Навигация:** [← README](../README.md) | [← Воркшоп 3](workshop-03-cycles-skills-checks.md) | [Реклама и запуск](../ai-agents-seller-workshop-promo-and-next-steps.md)

## Суть

Этот воркшоп про следующий уровень: как подключать внешние инструменты и источники к AI-агенту, работать с карточками товаров, изображениями, описаниями и проектировать интеграции для Wildberries / Ozon / Google Sheets / Telegram.

Главный результат:

```text
У участника появляется архитектура своей интеграции: что подключать, какие данные брать, куда сохранять, как агент будет этим пользоваться.
```

## Что показываем

### 1. Интеграционная карта

Показываем, что автоматизация состоит из связок:

```text
Marketplace API -> local script -> data/raw -> normalization -> context -> AI check -> Google Sheet / report / Telegram
```

Или:

```text
Карточка товара -> изображения / описание / отзывы -> product-profile.md -> AI-review -> ТЗ
```

Практика:

Каждый участник выбирает одну интеграционную цепочку.

### 2. MCP простым языком

MCP-сервер - это способ дать AI-агенту инструмент: например, получить данные из базы, прочитать Google Sheet, сходить в API или выполнить внутреннюю команду.

Не нужно грузить участников протоколом. Важно объяснить идею:

```text
Если агенту нужно не просто читать файлы, а ходить в внешний сервис, мы можем дать ему инструмент через MCP или отдельный локальный скрипт.
```

Примеры инструментов:

- `get_products`;
- `get_sales_report`;
- `get_stock_levels`;
- `update_google_sheet`;
- `download_product_card`;
- `save_weekly_report`;
- `send_telegram_summary`.

### 3. Плагин / локальный инструмент для маркетплейса

Идея:

```text
Плагин для Wildberries / Ozon - это набор инструкций и инструментов, которые знают:
- как получить данные;
- как сохранить их;
- как привести к нужному формату;
- какие проверки запустить;
- какой отчет сформировать.
```

Структура:

```text
integrations/
  wildberries/
    README.md
    api-map.md
    .env.example
    fetch-products.md
    fetch-sales.md
    fetch-stocks.md
    normalize-data.md
    checks.md
```

### 4. Работа с карточками товаров

Цепочка:

```text
ссылка / артикул -> данные карточки -> изображения -> описание -> отзывы -> product-profile.md -> AI-review -> ТЗ
```

Что можно сохранять:

```text
products/
  SKU123/
    product-profile.md
    images/
    reviews.md
    competitors.md
    card-audit.md
    designer-brief.md
```

### 5. Работа с изображениями

Что показываем:

- как прикладывать изображения карточек агенту;
- как просить описать, что видно;
- как проверять первый слайд;
- как готовить ТЗ для нового визуала;
- как хранить результаты в `product-profile.md`.

Промпт:

```text
Посмотри изображение первого слайда карточки.
Опиши, что покупатель понимает за 3 секунды.
Найди слабые места и предложи, что изменить.
Не оценивай красоту, оценивай понятность, оффер и закрытие боли.
```

### 6. Google Sheets и Telegram как выход системы

AI-система должна не только думать, но и отдавать результат туда, где работает команда.

Выходы:

- Google Sheet с findings;
- Markdown-отчет;
- Telegram-сводка;
- задача менеджеру;
- ТЗ дизайнеру;
- список SKU для проверки.

## Что делают участники

1. Выбирают одну интеграцию.
2. Описывают входные данные.
3. Описывают, где хранить результат.
4. Проектируют инструмент или MCP-команду.
5. Создают структуру папок.
6. Создают `api-map.md` или `integration-map.md`.
7. Создают профиль одного товара.
8. Описывают, какой отчет или действие должен получить собственник.

## Варианты интеграционных проектов

- Wildberries data tool;
- Google Sheets dashboard;
- Product card collector;
- Telegram owner summary;
- SQLite / PostgreSQL storage;
- MCP-инструмент для внутренних данных.

## Артефакты

- `integration-map.md`;
- структура папки интеграции;
- карта API или карта Google Sheets;
- профиль товара;
- сценарий MCP-инструмента или локального скрипта;
- план внедрения первой интеграции.

## Домашнее задание

1. Выбрать одну интеграцию для внедрения.
2. Доработать карту API / данных.
3. Создать тестовый запуск на демо-данных.
4. Сохранить результат в Markdown или Google Sheet.
5. Определить, что можно автоматизировать следующим циклом.
